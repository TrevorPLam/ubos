# AI Governance Layer

## Purpose

This directory contains the complete AI governance system for the UBOS marketing monorepo. It provides agents with the architectural rules, patterns, and constraints needed to maintain consistency, security, and performance across all development work.

## How Agents Should Use This Layer

1. **Start with AGENT_SYSTEM.md** - Read the constitutional rules first
2. **Reference patterns** - Use `/ai/patterns/` for implementation guidance
3. **Follow checklists** - Run `/ai/checklists/` before finalizing work
4. **Use examples** - Mirror `/ai/examples/` for canonical implementations
5. **Consult doctrine** - Reference `/ai/performance/`, `/ai/security/`, `/ai/design/` for specific requirements

## Priority Order of Documents

1. **AGENT_SYSTEM.md** - Constitutional rules (highest priority)
2. **Next.js God Tier Reference** - Production-grade Next.js patterns
3. **Patterns** - Implementation patterns for common scenarios
4. **Checklists** - Validation requirements before completion
5. **Examples** - Canonical implementations to mirror
6. **Doctrine** - Performance, security, design requirements
7. **Marketing Intelligence** - Conversion-optimized content guidelines
8. **Testing Doctrine** - Testing requirements and patterns
9. **Decision Logs** - Historical architectural decisions
10. **Prompt Library** - Reusable AI instructions
