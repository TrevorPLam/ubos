## ğŸ¯ Archive Policy

### Documents Moved to Archive When:

1. **Superseded**: Replaced by newer documentation
   - Old security policies replaced by updated versions
   - Previous compliance frameworks no longer relevant

2. **Deprecated**: No longer applicable to current system
   - Security controls for removed features
   - Old architecture security docs

3. **Historical**: Completed point-in-time assessments
   - Completed penetration test reports
   - Past security audits
   - Completed security projects

4. **Compliance Retention**: Required for audit trail
   - Previous SOC2 reports (retain 7 years)
   - Historical incident reports (retain per policy)
   - Old risk assessments (retain for comparison)
